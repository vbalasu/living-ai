# OpenClaw vs. Living-AI Properties

| Property | OpenClaw |
|---|---|
| Own compute | ✓ Local daemon / VPS |
| External stimulus | ✓ WhatsApp, Telegram, Slack, Discord, iMessage, Signal, Google Chat |
| Memories | ~ Markdown files on local disk (not clearly tri-partite) |
| Identity | ✓ Personal assistant persona |
| Money / transact | ✗ Not built in |
| Goals | ~ Task execution, less clear on long-running goals |
| Heartbeat / initiative | ✓ Heartbeat daemon — acts without prompting |
| Manages own state | ✓ Local-first |
| Multiple channels | ✓ Core feature |
| Multiple modalities | ~ Mostly text via messaging |
| Skills / tools | ✓ LLM-powered runtime takes real actions |

Closer to the living-ai vision than most agent frameworks — the heartbeat daemon + multi-channel inbox + local memory hits properties that AutoGPT-style agents miss. Main gaps: **money/transactions** and **multi-modality** (text-first).
